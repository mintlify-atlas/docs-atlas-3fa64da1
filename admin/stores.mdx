The store management system allows administrators to oversee partner stores where users can redeem their earned points for coupons and rewards.

## Store Data Model

Stores are stored in the `tb_tienda` table with the following structure:

### Store Fields

| Field | Type | Description |
|-------|------|-------------|
| `idTienda` | Integer | Unique store identifier (auto-generated) |
| `nomTienda` | String | Store name/business name |
| `usuario` | Usuario | Store manager account (FK) |
| `distrito` | Distrito | Store location district (FK) |

```java
@Entity
@Table(name = "tb_tienda")
public class Tienda {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    @Column(name="id_tienda")
    private Integer idTienda;

    @Column(name="nom_tienda")
    private String nomTienda;

    @ManyToOne
    @JoinColumn(name = "id_usu")
    private Usuario usuario;

    @ManyToOne
    @JoinColumn(name = "id_distrito")
    private Distrito distrito;
}
```

## Store Controller

The TiendaController manages all store operations:

```java
@RestController
@RequestMapping("/tienda")
```

Access store endpoints through the `/tienda` base path.

## Managing Stores

### Creating a New Store

<Steps>
  <Step title="Prepare Store Information">
    Gather the required details:
    
    <Check>Store/business name</Check>
    <Check>Store manager user account (must have "Tienda" role)</Check>
    <Check>District location</Check>
  </Step>
  
  <Step title="Create Store Manager Account">
    First, ensure a user account exists for the store manager:
    
    ```java
    // User must have Tienda role assigned
    Usuario storeManager = new Usuario();
    storeManager.setNomUsu("Manager Name");
    storeManager.setCorreoUsu("manager@store.com");
    storeManager.setRol(tiendaRole); // Tienda role
    storeManager.setActivo(true);
    ```
    
    <Note>
    Each store requires a dedicated user account with the "Tienda" role for management access.
    </Note>
  </Step>
  
  <Step title="Create Store Record">
    Create the store with all required associations:
    
    ```java
    Tienda newStore = new Tienda();
    newStore.setNomTienda("Green Market");
    newStore.setUsuario(storeManager);
    newStore.setDistrito(selectedDistrict);
    ```
  </Step>
  
  <Step title="Verify Store Creation">
    Confirm the store is accessible:
    - Store appears in store listings
    - Manager can log in with Tienda role
    - District association is correct
  </Step>
</Steps>

### Viewing Stores

<Tabs>
  <Tab title="All Stores">
    List all registered partner stores:
    
    ```sql
    SELECT 
      t.id_tienda,
      t.nom_tienda,
      d.nombre_distrito,
      u.nom_usu,
      u.correo_usu
    FROM tb_tienda t
    JOIN tb_distrito d ON t.id_distrito = d.id_distrito
    JOIN tb_usuario u ON t.id_usu = u.id_usu
    ORDER BY t.nom_tienda;
    ```
  </Tab>
  
  <Tab title="By District">
    Organize stores by geographic location:
    
    ```sql
    SELECT d.nombre_distrito, COUNT(*) as store_count
    FROM tb_tienda t
    JOIN tb_distrito d ON t.id_distrito = d.id_distrito
    GROUP BY d.nombre_distrito
    ORDER BY store_count DESC;
    ```
  </Tab>
  
  <Tab title="By Manager">
    View which users manage stores:
    
    ```sql
    SELECT 
      u.nom_usu,
      u.correo_usu,
      t.nom_tienda
    FROM tb_usuario u
    JOIN tb_tienda t ON u.id_usu = t.id_usu
    WHERE u.ROL = (SELECT id_rol FROM tb_rol WHERE descripcion = 'Tienda');
    ```
  </Tab>
</Tabs>

## Store-User Relationship

### Manager Association

Each store is linked to exactly one user account (the store manager):

```java
@ManyToOne
```

**Key Points:**
- One user can manage multiple stores
- Managers must have "Tienda" role
- Manager login grants access to store management features
- User's account settings affect store access

### Reporting Queries

```sql
-- Stores with most coupons
SELECT 
  t.nom_tienda,
  COUNT(c.id_cupon) as coupon_count
FROM tb_tienda t
LEFT JOIN tb_cupon c ON t.id_tienda = c.id_tienda
WHERE c.activo = true
GROUP BY t.id_tienda
ORDER BY coupon_count DESC;

-- Stores by district
SELECT 
  d.nombre_distrito,
  COUNT(t.id_tienda) as store_count
FROM tb_distrito d
LEFT JOIN tb_tienda t ON d.id_distrito = t.id_distrito
GROUP BY d.id_distrito
ORDER BY store_count DESC;

-- Most popular stores (by redemptions)
SELECT 
  t.nom_tienda,
  COUNT(uc.id_usu_cupon) as redemption_count
FROM tb_tienda t
JOIN tb_cupon c ON t.id_tienda = c.id_tienda
JOIN tb_usuario_cupon uc ON c.id_cupon = uc.id_cupon
GROUP BY t.id_tienda
ORDER BY redemption_count DESC
LIMIT 10;
```

## Managing Store Access

### Store Manager Login

Store managers access their dashboard through:

1. **Login** at `/auth/login` with their credentials
2. **Role Verification** - System confirms "Tienda" role
3. **Redirect** to `/tienda` route for store management
4. **Store Context** - Automatically loads their associated store(s)

```typescript
// Store module routing
{
  path: 'tienda',
  loadChildren: () =>
    import('./tienda/tienda.module').then((m) => m.TiendaModule)
}
```

### Access Control

<Warning>
Store managers should only access their own store data. Implement proper access controls:
</Warning>

- Filter store data by logged-in user's ID
- Prevent access to other stores' information
- Validate ownership before updates
- Audit all store modifications

## Store Deactivation

If a store needs to be removed from the platform:

<Steps>
  <Step title="Deactivate Associated Coupons">
    Set all store coupons to inactive:
    
    ```sql
    UPDATE tb_cupon
    SET activo = false
    WHERE id_tienda = ?;
    ```
  </Step>
  
  <Step title="Notify Users">
    Inform users who have coupons from this store:
    - Send notifications about closure
    - Provide alternative redemption options
    - Extend expiration dates if needed
  </Step>
  
  <Step title="Deactivate Manager Account">
    Disable the store manager's account:
    
    ```java
    storeManager.setActivo(false);
    ```
  </Step>
  
  <Step title="Archive Store Record">
    Consider soft delete instead of hard delete:
    - Maintain historical data
    - Preserve redemption history
    - Keep analytics intact
  </Step>
</Steps>

<Note>
Avoid deleting store records to preserve historical redemption data and analytics.
</Note>
