## Overview

GreenGuard's gamification system transforms environmental activism into an engaging, rewarding experience. Earn points for reporting incidents, climb community leaderboards, and redeem rewards that benefit both you and local businesses.

## Points System

### How Points Work

Every user account maintains a cumulative points balance:

```java
@Entity
@Table(name = "tb_usuario")
public class Usuario {
    @Column(name="puntos_usu")
    private int puntosUsu;  // Total lifetime points earned
    
    @Column(name="registro_usu")
    private LocalDateTime registroUsu;  // Account creation date
}
```

**Key Characteristics:**
- Points accumulate over your account lifetime
- Never expire (unless account is inactive)
- Visible on your profile dashboard
- Used as currency in the rewards marketplace
- Factor into community leaderboard rankings

### Point Calculation Logic

While the exact implementation may vary, the typical flow is:

1. **Report Submitted** (PE status) - 0 points
2. **Report Validated** (EP status) - +50 base points awarded
3. **Municipality Reviews** - Quality assessment may add bonus
4. **Issue Resolved** (RE status) - +100 completion bonus awarded
5. **Total Updated** - `puntosUsu` incremented with final total

```java
public void awardReportPoints(Reporte reporte) {
    Usuario usuario = reporte.getUsuario();
    int pointsToAward = 0;
    
    // Base points for validated report
    if (reporte.getEstado() == EstadoReporte.EP) {
        pointsToAward += 50;
    }
    
    // Bonus for resolution
    if (reporte.getEstado() == EstadoReporte.RE) {
        pointsToAward += 100;
        
        // Quality bonus for complete reports
        if (isQualityReport(reporte)) {
            pointsToAward += 25;
        }
        
        // Priority bonus
        if (isHighPriority(reporte.getTipoClasificacion())) {
            pointsToAward += 50;
        }
    }
    
    // Update user's total points
    usuario.setPuntosUsu(usuario.getPuntosUsu() + pointsToAward);
}
```

## Spending Points

Points serve as currency in the GreenGuard marketplace. When you redeem a coupon:

```java
@Entity
@Table(name = "tb_cupon")
public class Cupon {
    @Column(name="puntos_requeridos")
    private int puntosRequeridos;  // Points needed to redeem
}
```

The redemption process:

1. Browse available coupons in marketplace
2. Check if you have sufficient `puntosUsu` ≥ `puntosRequeridos`
3. Redeem coupon - points are deducted from your balance
4. Receive coupon code for use at partner stores

<Warning>
  Points are **deducted immediately** upon coupon redemption. Make sure you plan to use the coupon before redeeming.
</Warning>

Maintain consistency to earn streak bonuses:

Streak achievements award bonus points and special badges.

## Community Leaderboards

### How Rankings Work

Leaderboards showcase top contributors in various categories:

**Global Leaderboard**
- Ranked by total `puntosUsu` across all users
- Updated in real-time as points are awarded
- Top 100 contributors displayed

**District Leaderboard**
- Filtered by `distrito` where reports are filed
- Encourages local community engagement
- Monthly resets to give everyone a fresh start

**Monthly Leaderboard**
- Points earned in current calendar month only
- Resets on the 1st of each month
- Winner receives special recognition

### Leaderboard Query Example

```sql
-- Top 10 users by total points
SELECT 
    nom_usu,
    ape_pat_usu,
    puntos_usu,
    ROW_NUMBER() OVER (ORDER BY puntos_usu DESC) as rank
FROM tb_usuario
WHERE activo = true
ORDER BY puntos_usu DESC
LIMIT 10;
```

## Fairness & Integrity

### Anti-Fraud Measures

GreenGuard maintains system integrity through:

- **Report Validation** - All reports reviewed before awarding points
- **Duplicate Detection** - Similar reports flagged to prevent gaming
- **Photo Verification** - Images checked for authenticity
- **Location Verification** - GPS coordinates validated against district boundaries
- **Rate Limiting** - Maximum reports per user per day
- **Manual Review** - Suspicious patterns flagged for administrator review

<Warning>
  Fraudulent reporting results in:
  - Point forfeiture
  - Account suspension
  - Ban from rewards marketplace
  - Potential legal consequences for false reports
</Warning>
