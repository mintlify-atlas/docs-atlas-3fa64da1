## Welcome to GreenGuard

GreenGuard is a civic engagement platform that empowers citizens to report environmental incidents and earn rewards for making their communities cleaner and safer. Follow this guide to create your account and submit your first incident report.

<Steps>
  <Step title="Create Your Account">
    Start by registering a new user account on the GreenGuard platform.

    **Required Information:**
    - Full name (first name, paternal surname, maternal surname)
    - Valid email address
    - Strong password
    - Phone number
    - Document ID (DNI or identification number)
    - Gender
    - Profile image (optional)

    **Registration Endpoint:**
    ```http
    POST /auth/register
    ```

    The system automatically assigns you the "Cliente" (Citizen) role and initializes your account with:
    - **0 points** to start earning through incident reports
    - Active account status
    - Registration timestamp
    - Optional FCM token for push notifications

    <Note>
      Your account is activated immediately upon registration. You can start reporting incidents right away!
    </Note>
  </Step>

  <Step title="Complete Your Profile">
    After registration, enhance your profile to get the most out of GreenGuard:

    1. **Upload a profile photo** to personalize your account
    2. **Enable push notifications** by allowing FCM token registration
    3. **Verify your contact information** to receive updates about your reports

    The system stores your FCM (Firebase Cloud Messaging) token for real-time notifications about:
    - Report status updates
    - Available rewards
    - Community achievements
  </Step>

  <Step title="Submit Your First Incident Report">
    Now you're ready to make an impact! Report environmental incidents in your community.

    ### What You Can Report

    Common incident types include:
    - Illegal dumping
    - Accumulated trash
    - Abandoned vehicles
    - Damaged infrastructure
    - Environmental hazards
    - Public space issues

    ### Creating a Report

    Each report requires:

    <CardGroup cols={2}>
      <Card title="Location Data" icon="map-pin">
        - GPS coordinates (latitude/longitude)
        - District selection
        - Precise incident location
      </Card>
      <Card title="Incident Details" icon="clipboard">
        - Incident type classification
        - Detailed description
        - Photo evidence
        - Severity classification
      </Card>
    </CardGroup>

    **Report Model Structure:**
    ```java
    @Entity
    @Table(name = "tb_reporte")
    public class Reporte {
        private String numReport;         // Auto-generated report number
        private Usuario usuario;          // Your user account
        private String detalleRepo;       // Incident description
        private String imagenRepo;        // Photo URL
        private EstadoReporte estado;     // Status: PE, EP, RE, CA
        private BigDecimal latitud;       // GPS latitude
        private BigDecimal longitud;      // GPS longitude
        private TipoIncidentes tipoIncidente;     // Incident category
        private TipoClasificacion tipoClasificacion; // Severity level
        private Distrito distrito;        // District/area
        private LocalDateTime repoRegistado;  // Submission timestamp
        private LocalDateTime repoResuelto;   // Resolution timestamp
    }
    ```

    ### Report Status Lifecycle

    Your report progresses through these states:
    - **PE (Pendiente)** - Submitted and awaiting review
    - **EP (En Proceso)** - Municipality is working on it
    - **RE (Resuelto)** - Issue resolved
    - **CA (Cancelado)** - Report cancelled or invalid

    <Warning>
      Always include a clear photo of the incident. Reports with photo evidence are processed faster by municipal authorities.
    </Warning>
  </Step>

  <Step title="Track Your Report">
    After submitting your report, you'll receive:

    1. **A unique report number** for tracking
    2. **Push notification** confirming receipt
    3. **Status updates** as the municipality processes your report

    You can monitor:
    - Current report status (Pending, In Process, Resolved, Cancelled)
    - Timestamp of status changes
    - Municipal response notes
    - Resolution timeline

    <Note>
      You'll receive a push notification each time your report status changes. Make sure notifications are enabled!
    </Note>
  </Step>

  <Step title="Earn Points and Rewards">
    Congratulations! You've submitted your first report. Now you can:

    - **Earn points** when your report is validated and processed
    - **Track your impact** through your cumulative points balance
    - **Redeem rewards** in the marketplace once you accumulate enough points
    - **Climb the leaderboard** and become a top contributor

    Points are stored in your user profile:
    ```java
    @Column(name="puntos_usu")
    private int puntosUsu;  // Your total points balance
    ```

    Visit the [Rewards Marketplace](/features/rewards) to browse available coupons and see how many points you need to redeem them.
  </Step>
</Steps>
