api: 'PUT /usuario/{id}'
---

## Overview

This endpoint allows updating a user's profile information. Only the fields provided in the request body will be updated.

## Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the user to update
</ParamField>

## Request Body

<ParamField body="nomUsu" type="string">
  User's first name
</ParamField>

<ParamField body="apePatUsu" type="string">
  User's paternal surname
</ParamField>

<ParamField body="apeMatUsu" type="string">
  User's maternal surname
</ParamField>

<ParamField body="documentoUsu" type="string">
  User's identification document number
</ParamField>

<ParamField body="correoUsu" type="string">
  User's email address (must be unique)
</ParamField>

<ParamField body="telefonoUsu" type="string">
  User's phone number
</ParamField>

<ParamField body="generoUsu" type="string">
  User's gender
</ParamField>

<ParamField body="imagenUSU" type="string">
  URL or path to user's profile image
</ParamField>

<ParamField body="fmcToken" type="string">
  Firebase Cloud Messaging token for push notifications
</ParamField>

<ParamField body="activo" type="boolean">
  Whether the user account is active (admin only)
</ParamField>

## Example Request

```bash
curl -X PUT "https://api.greenguard.com/usuario/123" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "telefonoUsu": "+51999888777",
    "imagenUSU": "/images/profiles/123_updated.jpg"
  }'
```

## Update FCM Token

To update only the Firebase Cloud Messaging token (common for mobile apps):

```bash
curl -X PUT "https://api.greenguard.com/usuario/123" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "fmcToken": "new_fcm_token_here"
  }'
```

## Protected Fields

The following fields cannot be updated through this endpoint:
- `idUsu` - User ID is immutable
- `registroUsu` - Registration date is set only once
- `puntosUsu` - Use the points management endpoint instead
- `passwordUsu` - Use the password change endpoint instead
- `rol` - Role changes require admin privileges via separate endpoint
- `fmcTokenFecha` - Automatically updated when fmcToken changes

## Validation Rules

- **correoUsu**: Must be a valid email format and unique across all users
- **documentoUsu**: Should follow the document format for the user's country
- **telefonoUsu**: Should include country code (e.g., +51 for Peru)
- **generoUsu**: Typically accepts standard gender codes
- **imagenUSU**: Should be a valid URL or file path
