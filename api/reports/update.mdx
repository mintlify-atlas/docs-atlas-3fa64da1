description: 'Update an existing incident report'

---

## Overview

Update an existing incident report. This endpoint allows modifying report details, status, location, and associated classifications. The `repoResuelto` timestamp is automatically set when status changes to `RE` (Resolved).

## Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the report to update
</ParamField>

## Request Body

<ParamField body="detalleRepo" type="string">
  Updated description of the incident
</ParamField>

<ParamField body="imagenRepo" type="string">
  Updated image URL or path
</ParamField>

<ParamField body="estado" type="string">
  Updated report status:
  - `PE` - Pendiente (Pending)
  - `EP` - En Proceso (In Process)
  - `RE` - Resuelto (Resolved)
  - `CA` - Cancelado (Cancelled)
</ParamField>

<ParamField body="latitud" type="decimal">
  Updated latitude coordinate
</ParamField>

<ParamField body="longitud" type="decimal">
  Updated longitude coordinate
</ParamField>

<ParamField body="idTipoInci" type="integer">
  Updated incident type ID
</ParamField>

<ParamField body="idTipoClasi" type="integer">
  Updated classification type ID
</ParamField>

<ParamField body="idDistrito" type="integer">
  Updated district ID
</ParamField>

<Note>
  You only need to include the fields you want to update. Omitted fields will remain unchanged.
</Note>

## Example Requests

### Update report status to In Process

```bash
curl -X PUT https://api.greenguard.com/reporte/123 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "estado": "EP"
  }'
```

### Update report status to Resolved

```bash
curl -X PUT https://api.greenguard.com/reporte/123 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "estado": "RE"
  }'
```

### Update multiple fields

```bash
curl -X PUT https://api.greenguard.com/reporte/123 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "detalleRepo": "Basura acumulada - Actualizado: Ya fue limpiado parcialmente",
    "estado": "EP",
    "idTipoClasi": 2
  }'
```

### Update location coordinates

```bash
curl -X PUT https://api.greenguard.com/reporte/123 \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{
    "latitud": -12.0465,
    "longitud": -77.0429
  }'
```

## Status Transitions

The following status transitions are common:

- `PE` (Pending) → `EP` (In Process): When authorities start working on the report
- `EP` (In Process) → `RE` (Resolved): When the incident is resolved
- `PE` or `EP` → `CA` (Cancelled): When the report is invalid or cancelled
- `RE` (Resolved): Final state, typically not changed after this

<Warning>
  When updating a report to `RE` (Resolved) status, the `repoResuelto` timestamp is automatically set to the current date and time.
</Warning>
