## Overview

The login endpoint authenticates existing users using their email and password credentials. Upon successful authentication, user information is returned including their profile details and role.

## Endpoint

```
POST /auth/login
```

## Request Body

<ParamField body="correoUsu" type="string" required>
  User's email address. Must match an existing account in the system.
  
  Example: `usuario@example.com`
</ParamField>

<ParamField body="passwordUsu" type="string" required>
  User's password. This will be validated against the encrypted password stored in the database.
  
  Example: `SecurePassword123!`
</ParamField>

## Response

<ResponseField name="idUsu" type="integer">
  Unique identifier for the user
</ResponseField>

<ResponseField name="nomUsu" type="string">
  User's first name
</ResponseField>

<ResponseField name="apePatUsu" type="string">
  User's paternal surname
</ResponseField>

<ResponseField name="apeMatUsu" type="string">
  User's maternal surname
</ResponseField>

<ResponseField name="documentoUsu" type="string">
  User's identification document number
</ResponseField>

<ResponseField name="correoUsu" type="string">
  User's email address
</ResponseField>

<ResponseField name="telefonoUsu" type="string">
  User's phone number
</ResponseField>

<ResponseField name="generoUsu" type="string">
  User's gender
</ResponseField>

<ResponseField name="imagenUSU" type="string">
  URL or path to user's profile image
</ResponseField>

<ResponseField name="registroUsu" type="datetime">
  Timestamp of when the user registered
</ResponseField>

<ResponseField name="puntosUsu" type="integer">
  User's accumulated reward points
</ResponseField>

<ResponseField name="rol" type="object">
  User's role information
  
  <Expandable title="rol properties">
    <ResponseField name="idRol" type="integer">
      Role identifier
    </ResponseField>
    
    <ResponseField name="descripcion" type="string">
      Role description (e.g., "User", "Admin", "Moderator")
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="activo" type="boolean">
  Whether the user account is active
</ResponseField>

<ResponseField name="fmcToken" type="string">
  Firebase Cloud Messaging token for push notifications
</ResponseField>

<ResponseField name="fmcTokenFecha" type="datetime">
  Timestamp of when the FCM token was created
</ResponseField>

## Example Request

<CodeGroup>

```bash cURL
curl -X POST https://api.greenguard.com/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "correoUsu": "juan.perez@example.com",
    "passwordUsu": "SecurePassword123!"
  }'
```

```javascript JavaScript
const response = await fetch('https://api.greenguard.com/auth/login', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify({
    correoUsu: 'juan.perez@example.com',
    passwordUsu: 'SecurePassword123!'
  })
});

const data = await response.json();
console.log(data);
```

```python Python
import requests

url = "https://api.greenguard.com/auth/login"
payload = {
    "correoUsu": "juan.perez@example.com",
    "passwordUsu": "SecurePassword123!"
}

response = requests.post(url, json=payload)
data = response.json()
print(data)
```

```java Java
import java.net.http.*;
import java.net.URI;

HttpClient client = HttpClient.newHttpClient();
String json = """
{
  "correoUsu": "juan.perez@example.com",
  "passwordUsu": "SecurePassword123!"
}
""";

HttpRequest request = HttpRequest.newBuilder()
    .uri(URI.create("https://api.greenguard.com/auth/login"))
    .header("Content-Type", "application/json")
    .POST(HttpRequest.BodyPublishers.ofString(json))
    .build();

HttpResponse<String> response = client.send(request, 
    HttpResponse.BodyHandlers.ofString());
System.out.println(response.body());
```

## Response Codes

| Code | Description |
|------|-------------|
| 200 | Login successful - user authenticated |
| 400 | Bad request - missing or invalid parameters |
| 401 | Unauthorized - invalid email or password |
| 403 | Forbidden - account is deactivated |
| 500 | Internal server error |

## Notes

- The password is validated against the encrypted value stored in the `password_usu` field of the `tb_usuario` table
- Inactive accounts (where `activo` is false) cannot log in
- The response includes the complete user profile, including role information
- The `fmcToken` field may be null if the user hasn't registered for push notifications
- Failed login attempts should be rate-limited to prevent brute force attacks

## Security

<Warning>
  Always use HTTPS when transmitting login credentials. Never send passwords over unencrypted connections.
</Warning>

<Info>
  Consider implementing additional security measures such as:
  - Rate limiting on failed login attempts
  - Two-factor authentication for sensitive accounts
  - Session token management
  - Password complexity requirements
</Info>
